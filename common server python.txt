## CommandResults

CommandResults class - use to return results to warroom
**Arguments** :
i. outputs_prefix _str_ : should be identical to the prefix in the yml contextPath in yml file. for
example: CortexXDR.Incident
ii. outputs_key_field _str or list[str]_ : primary key field in the main object. If the command returns
Incidents, and of the properties of Incident is incident_id, then outputs_key_field='incident_id'. If
object has multiple unique keys, then list of strings is supported outputs_key_field=['id1', 'id2']
iii. outputs _list or dict_ : the data to be returned and will be set to context
iv. indicators _list_ : DEPRECATED: use 'indicator' instead.
v. indicator _Common.Indicator_ : single indicator like Common.IP, Common.URL, Common.File,
etc.
vi. readable_output _str_ : (Optional) markdown string that will be presented in the warroom, should
be human readable - (HumanReadable) - if not set, readable output will be generated
vii. raw_response _dict | list_ : must be dictionary, if not provided then will be equal to outputs. usually
must be the original raw response from the 3rd party service (originally Contents)
viii. indicators_timeline _IndicatorsTimeline_ : must be an IndicatorsTimeline. used by the server to
populate an indicator's timeline.
ix. ignore_auto_extract _bool_ : must be a boolean, default value is False. Used to prevent
AutoExtract on output.
x. relationships _list of EntityRelationship_ : List of relationships of the indicator.

```
| BarColumnPieWidget.to_display()
```

xi. mark_as_note _bool_ : must be a boolean, default value is False. Used to mark entry as note.
xii. tags _list_ : must be a list, default value is None. Used to tag war room entries.
xiii. entry_type _int code of EntryType_ : type of return value, see EntryType
xiv. scheduled_command _ScheduledCommand_ : manages the way the command should be polled.
xv. execution_metrics _ExecutionMetrics_ : contains metric data about a command's execution
xvi. replace_existing _bool_ : Replace the context value at outputs_prefix if it exists. Works only if
outputs_prefix is a path to a nested value i.e., contains a period. For example, the "next token"
result should always be overwritten. This response can be returned as follows:

## PollResult

The response object for polling functions. This object contains information about whether to run
again, and what the CommandResults are in case of success, or failure.
**Returns** :
i. PollResult - PollResult
Constructor for PollResult
**Arguments** :
i. response _Any_ : The response of the command in the event of success, or in case of failure but
Polling is false
ii. continue_to_poll _Union[bool, Callable]_ : An iterable of relevant keys list from the json. Notice
